# UX/UI Testing (Heuristic Evaluation)

## ¿Qué es?
El **UX/UI Testing** se centra en evaluar la experiencia del usuario y la interfaz gráfica de un sistema.  
A diferencia del QA funcional (que valida si el software funciona), aquí se analiza **qué tan fácil, intuitivo y agradable es usarlo**.  
Una técnica clave es la **Evaluación Heurística**, basada en principios de usabilidad (como las 10 heurísticas de Jakob Nielsen).

## ¿Qué función cumple?
- Detectar problemas de usabilidad antes de que lleguen a los usuarios finales.
- Mejorar la eficiencia, satisfacción y accesibilidad de la interfaz.
- Complementar el testing funcional: un sistema puede funcionar bien pero ser difícil de usar.
- Asegurar que el diseño cumpla con estándares de UX y buenas prácticas.

## Heurísticas más comunes (Nielsen)
1. **Visibilidad del estado del sistema:** el usuario siempre sabe qué está pasando.  
2. **Correspondencia entre sistema y mundo real:** usar lenguaje y metáforas familiares.  
3. **Control y libertad del usuario:** permitir deshacer y rehacer acciones.  
4. **Consistencia y estándares:** elementos que se comportan de forma predecible.  
5. **Prevención de errores:** diseño que evita equivocaciones.  
6. **Reconocimiento mejor que recuerdo:** opciones visibles, no depender de memoria.  
7. **Flexibilidad y eficiencia:** accesos rápidos para usuarios expertos.  
8. **Estética y diseño minimalista:** sin ruido visual innecesario.  
9. **Ayuda para reconocer, diagnosticar y recuperar errores.**  
10. **Ayuda y documentación accesible.**

## Herramientas comunes
- **Figma / Sketch / Adobe XD:** para prototipos y diseño de interfaces.  
- **Maze / UsabilityHub / Hotjar:** para pruebas con usuarios reales.  
- **Checklist de heurísticas:** para evaluaciones rápidas sin usuarios.  
- **Google Forms / Excel:** para registrar hallazgos y feedback.  

## Cómo se usa
1. Se define el alcance: qué pantallas o flujos se van a evaluar.  
2. Se aplica la lista de heurísticas a cada elemento de la interfaz.  
3. Se documentan problemas encontrados, su severidad y recomendaciones.  
4. Se entregan reportes visuales o tablas con hallazgos.  
5. Se priorizan mejoras junto al equipo de diseño y desarrollo.  

## Ejemplo
**Evaluación de pantalla de login**  
- **Problema:** No hay feedback cuando el usuario ingresa una contraseña incorrecta.  
- **Heurística violada:** Visibilidad del estado del sistema.  
- **Recomendación:** Mostrar mensaje claro de error (“Contraseña incorrecta, intente nuevamente”).  

## Aplicaciones
- Testing de aplicaciones web y móviles.  
- Evaluación de prototipos antes del desarrollo.  
- Auditorías de usabilidad en sistemas existentes.  
- Mejora continua de productos digitales.
